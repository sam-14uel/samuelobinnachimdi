AfriWallStreet

Project Overview

AfriWallStreet is a comprehensive social media platform designed specifically for the African market. The platform incorporates a wide range of features to provide users with a seamless and engaging social experience.

Key Features

  • Real-time Chat: Implemented using Django Channels and WebSockets to enable instant messaging between users.
  • Online/Offline Tracking: Real-time status updates showing when users are active on the platform.
  • AI Chatbot: Integrated intelligent assistant that helps users navigate the platform and provides automated support.
  • Advanced Search: Powerful search functionality allowing users to find content, users, and topics quickly.
  • Posting Features: Rich media posting capabilities, including text, images, videos, and links.
  • Notifications: Real-time notification system for interactions, messages, and platform updates.

Technologies Used

  • Backend: Django framework with Django REST Framework for API endpoints
  • Database: PostgreSQL for robust data storage and complex queries
  • Caching: Redis for improved performance and real-time features
  • WebSockets: Django Channels for real-time communication
  • Server: Daphne ASGI server for handling both HTTP and WebSocket protocols
  • Frontend: HTML, CSS, JavaScript with reactive components

Challenges & Solutions

One of the main challenges was implementing a scalable real-time messaging system. I solved this by using Django Channels with Redis as the channel layer, allowing for efficient message broadcasting and private messaging capabilities that could scale with user growth.

Agent Tool Protocol (ATP)

Project Overview

The Agent Tool Protocol (ATP) is a lightweight but powerful protocol that allows Large Language Models (LLMs) to call secure, developer-built tools called ToolKits. ATP enables real-world execution of tasks like API access, web scraping, automation, and more, turning static models into capable AI agents.

Key Features

  • ToolKit SDK: Python SDK to define and register tools using decorators
  • Secure Execution: Local or deployed ToolKits that are callable by AI
  • LLM-Agnostic: Works with GPT, Claude, Llama, and other models
  • Developer-Oriented: Lightweight, composable, open by design
  • Protocol Layer: Designed to integrate with any AI infrastructure

Technologies Used

  • Language: Python
  • Packaging: PyPI SDK (`atp`)
  • Architecture: Local and remote ToolKit invocation with metadata sync

Use Cases

ATP has been used to build developer agents, AI productivity assistants, scraping bots, system controllers, and more. It's the core protocol that powers the ChatATP platform and supports scalable agent ecosystems.

ChatATP

Project Overview

ChatATP is a cross-platform AI Agent platform where users create autonomous agents by combining their preferred LLMs with ToolKits. It supports real-time execution, cloud syncing, ToolKit management, and user-controlled collections — built for developers, businesses, and power users.

Key Features

  • ToolKit Collection: Users can add/remove tools to modify their agent’s abilities
  • Cloud Sync: Agent state and tools are synced across devices
  • LLM Integration: Plug in GPT, Claude, Llama, Grok, etc.
  • Real-Time Execution: Powered by WebSockets and background jobs
  • Private & Public Tools: Businesses can build internal ToolKits or publish them

Technologies Used

  • Frontend: React, Vite, TypeScript
  • Backend: Django, Django REST Framework
  • Real-Time: Django Channels, Daphne, Redis
  • Database: PostgreSQL (hosted on Neon)
  • LLM APIs: OpenRouter, OpenAI, Anthropic

Vision

ChatATP represents the shift from chat interfaces to fully programmable agents. It’s built to scale as an agent protocol platform, enabling no-code agent creation, business AI tools, and a developer-driven ToolKitStore.
